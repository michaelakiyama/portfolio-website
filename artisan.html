Artisan

Artisan is a responsive ecommerce web app that highlights the artistry of hand made household products and the stories of the artisans who brought them to life.

My Process

Research

It's important to fully understand the issue that is trying to be solved in order to create a meaningful and useful solution that meets the needs of users. Time was spent to clarify these project components before starting.

Userflow

Now that the above parameters were defined, a userflow diagram was generated to visualize the steps required to complete various user tasks.

Brand Development

With a rough concept created and a user roadmap planned out, I could now turn my attention to developing the brand, desired messaging and visual path ahead.

Low-Fi Wireframes

These visual elements now needed a framework. I enjoyed a small break from the digital world to create low-fi wireframes with pencil and paper.

Mid-Fi Wireframes

Low-fi wireframes combined with brand appropriate typefaces and icons made way for mid-fidelity sample screens.

User Testing

Test early, test often. Mid-fi wireframes offered an opportunity to collect feedback before continuing on with any layout, function and design decisions.

Design Changes

"Unsure about this screen? User could maybe click on the change view button and see the result right away?" Changes were made from user input.

More Testing

Test early, test often. One more check in with users was needed to help with some background visual decisions. I was exploring an alternative to pure white which is effective but common.

More Changes

Test early, test often. Results were split between the two options with comments varying between better contrast with white and less eyestrain with color. Ultimately, I decided to use both.

Responsive Focus

Final designs were gradually coming together and next it was time to concentrated on desktop and tablet breakpoints.

Mockups

And finally, polished designs were presented through market ready mockups.

Next Steps

Additional testing and user interviews are needed to prepare for the next iteration. Are user needs being met? How easy are tasks accomplished? What features are missing? Is their still a competitive edge? These are just a few questions to be answered in the following stage of development.
